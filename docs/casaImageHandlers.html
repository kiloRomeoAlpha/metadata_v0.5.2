class CasaImageHandlers(pyrap.images.image.image)
    Class is subclassed off the pyrap.images.image class, which in turn
is subclassed from the Casacore Image class. Constructor opens a passed
CASA Image filename.
 
The Casacore Image class defines a number of methods to extract metadata
from the passed image, and is not overridden here.
 
Examples of such methods are 
 
name() - filename passed
ndim() - N dimensions of image
shape()- shape of image cube, pixels
size() - N_of pixels in image.
unit() - units of image, eg. 'Jy/beam'
 
Some calls will still return data structures that will have to be further
parsed by this class. 
 
eg.,
 
imageinfo()
 
{'restoringbeam': {
    'major': {'unit': 'arcsec', 'value': 1.3832687139511108},
    'positionangle': {'unit': 'deg', 'value': 33.8624153137207},
    'minor': {'unit': 'arcsec', 'value': 1.2961828708648682}
    },
'objectname': '3C129', 
'imagetype': 'Intensity'
}
 
which means 'objectname' is available as
 
imageinfo()['objectname']
 
'3C129'
 
Other methods will manifest from a call to receive the passed image's 
coordinate systems via
 
coordinates().get_telescope()
 
which is more efficiently extracted from an instance of image.coordinates,
 
i.e.
 
pimCoords = coordinates()
pimCoords.get_telescope()
 
For more documentation on available image methods see,
 
<http://www.atnf.csiro.au/computing/software/casacore/pyrap/docs/pyrap_images.html>
 
Still other metadata will need to be accessed directly from available data
structures but which do not have methods provided. Methods for those data
are provided as 'meta-methods', such as 'buildPointing()' will call these methods on the 
instances, gather and package the data in an appropriate way.

class CasaImageValueError(exceptions.AttributeError)
    A casaImageValueError will be raised when a CASA Image file contains
an unknown element as presented by a call to keywordnames().  A the top level,
CASA Images are presented as a list of string values, whose underlying contents
may be one of several types, either a table, or some other data structure, such
as a nested dictionary, or a simple string.  To data, only the string values of 
 
'logtable', 'coords', 'imageinfo', 'units'
 
are known.
 
Image list inclusions for image metadata, imageInclusions.listInclusions,
will only examine the expected keyword lists and strings,
 
'coords', 'imageinfo', 'units'
